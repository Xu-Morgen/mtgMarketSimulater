# 前端人工验收记录

本目录在用户可见前端轨道完成时保存 `<迭代ID>.md`，例如 `I06F.md`、`I12F.md`、`I30F.md`。记录只陈述已实际执行的浏览器验收，不预先创建“通过”文件，也不以自动化测试结果代替人工操作。

每份记录至少包含：

- 迭代 ID、执行日期、执行人、提交或构建标识。
- 浏览器与版本、桌面/窄屏视口、API/数据库测试环境。
- 使用的测试账号和数据标识；不得记录密码、令牌、Cookie 或密钥。
- 对应 FE-M 验收 ID、逐步操作、预期、实际结果与通过/失败状态。
- 截图或录屏的仓库外安全路径/受控附件标识，以及用于排障的请求 ID、用户/订单/活动等实体 ID。
- 失败项、风险和复测结果；存在失败步骤时不得勾选迭代完成。

建议单项格式：

```markdown
## FE-Mxx：场景名称

- 前置数据：
- 操作步骤：
- 预期结果：
- 实际结果：
- 证据：
- 结果：通过 / 失败
```

完整步骤和完成门禁以仓库根目录的《前端需求》和《完整项目迭代实施计划与检查清单》为准。
