# 测试层（tests）

本层集中放置前端测试及其测试夹具，生产目录不依赖本层。按测试目的组织为单元测试、组件/集成测试和端到端测试；测试文件的具体命名和运行器将在引入 Vitest、React Testing Library 与 Playwright 时确定。

应优先覆盖：API mutation 是否携带 idempotency key 并在成功后刷新查询；服务端返回的不可交易原因、下一次工作资金领取时间、开包重复提交、买单/卖单二次确认与比赛战报的加载/成功/模板降级状态。移动端关键流程在产品 I6 进入实施后纳入对应门禁；MVP 只要求窄屏下核心操作不被阻断。

测试可模拟 API 响应验证展示与交互，但不得在前端测试中复刻或断言客户端结算算法。余额、费用、保证金、价格、开包和赛果均以服务端契约返回值为准。
