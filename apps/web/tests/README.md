# 测试层（tests）

本层集中放置前端测试、人工验收记录及其测试夹具，生产目录不依赖本层。按测试目的组织为单元测试、组件/集成测试、Playwright 端到端测试和 `manual/` 人工验收；自动化文件的具体命名在 I06F 引入 React Testing Library 与 Playwright 时确定。

应优先覆盖：API mutation 是否携带 idempotency key 并在成功后刷新查询；服务端返回的不可交易原因、下一次工作资金领取时间、开包重复提交、买单/卖单二次确认与比赛战报的加载/成功/模板降级状态；管理员/普通玩家权限、活动版本发布、用户补偿命令和日志脱敏。移动端关键流程在产品 I6 进入实施后纳入对应门禁；MVP 只要求窄屏下核心操作不被阻断。

`manual/<迭代ID>.md` 每份记录日期、构建/提交标识、浏览器、测试账号/数据、逐步结果和截图/录屏路径；只创建模板不算通过。用户可见迭代只有在相关页面、Playwright 和人工记录均通过后才能完成，具体 FE-M01–FE-M09 步骤见根目录《前端需求》。

测试可模拟 API 响应验证展示与交互，但不得在前端测试中复刻或断言客户端结算算法。余额、费用、保证金、价格、开包和赛果均以服务端契约返回值为准。
