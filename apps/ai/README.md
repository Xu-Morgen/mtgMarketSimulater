# AI 赛事叙事项目

本项目是比赛叙事的受限旁路能力：它只消费**已经由服务端规则引擎结算**的比赛摘要，并产出经校验的 AI 或模板战报。它不是经济、库存、订单、开包、赛事结算或权限系统的权威。

## 架构原则

- 采用以领域与端口为中心的分层结构；依赖方向始终由外向内。
- 模型调用仅由基础设施适配器执行，且不配置函数工具、数据库工具或搜索工具。
- SQLite 任务、结果与审计由受控仓储端口接入；业务层不能直接拼接 SQL 或操作数据库连接。
- 任一失败均保存可审计原因，并返回模板战报；不得影响既有赛果、余额、库存、奖励或成就。
- 浏览器不得导入本项目、取得 API 密钥或直接调用模型。

详细目录、依赖规则和处理流见 [ARCHITECTURE.md](./ARCHITECTURE.md)。每个架构目录中的 `README.md` 是该目录的职责边界与后续实现准则。
