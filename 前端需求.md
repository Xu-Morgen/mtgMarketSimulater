# 前端需求

本文件是前端交付范围；架构选型与完整边界见[技术栈与模块职责边界.md](技术栈与模块职责边界.md)。前端只展示服务器真相和提交用户意图，不结算经济数据。

## 1. 技术与工程要求

- Next.js + React + TypeScript；Tailwind CSS；TanStack Query；Zustand 仅存 UI 临时状态；ECharts 与 Framer Motion。
- 所有 API 类型来自共享 `contracts` 包；不得手写与后端漂移的 DTO。
- 手机优先适配市场、库存、开包、比赛和每日工作资金流程。
- 所有变更型请求带 idempotency key；成功后以服务端响应刷新缓存。

## 2. 页面与功能

| 页面 | 必须能力 | 不能承担的能力 |
| --- | --- | --- |
| 登录与账户 | 注册、登录、退出、会话过期提示 | 计算权限、发钱或保存密码明文。 |
| 首页 | 余额、净资产、工作资金领取、今日比赛、市场指数、待领奖励 | 客户端累计每日资金或判断奖励资格。 |
| 补充包商店 | 补充包、概率、价格、购买和开包动画 | 抽卡、扣款或保底计算。 |
| 库存与图鉴 | SKU、数量、成本、双价格、市值、锁定状态、收藏进度 | 修改库存、成本、价格或锁定状态。 |
| 市场 | 搜索、筛选、Cardmarket 参考价与游戏内价双曲线、NPC 报价、玩家订单簿 | 生成报价、撮合、冻结资金或库存。 |
| 挂售确认 | 展示版本、数量、标价、手续费、预计到手、保证金与取消规则；二次确认 | 自行计算费用或绕过确认创建订单。 |
| 卡组与比赛 | 编辑草稿、显示合法性、报名、战报、成就 | 判定赛果、发奖或直接解锁成就。 |
| 管理后台 | 在权限校验后编辑内容、参数、赛事与查看日志 | 无审计修改生产数据。 |

## 3. 关键交互验收

1. 领取工作资金按钮在成功后立即禁用，并显示服务器返回的下一次可领时间。
2. 开包重复点击或网络重试不会重复展示为新的服务端结果。
3. 挂售必须经过确认页；确认页只显示后端预览值，不能直接编辑保证金或预计到手金额。
4. 无价、零价、映射异常或过期数据 SKU 显示不可新增交易原因。
5. 比赛赛果先显示确定性结果；AI 战报加载中、成功、模板降级三种状态清晰可见。
6. 所有价格界面显示数据来源与更新时间，不使用“Cardmarket 实时价格”表述。
