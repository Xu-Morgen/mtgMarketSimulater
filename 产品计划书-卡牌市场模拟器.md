# 卡牌市场模拟器：产品计划书（需求待分级）

## 1. 产品概述

### 产品定位

一款可快速部署到服务器的网页游戏。玩家通过初始资金、购买补充包、开包获得卡牌、在模拟市场中买卖卡牌、管理库存，并根据市场价格的上涨与下跌制定经营策略。游戏重点是“收藏乐趣 + 市场交易 + 风险决策”，不涉及现实货币或真实卡牌交易。

### 目标用户

- 喜欢集换式卡牌、开包和收藏的玩家。
- 喜欢价格波动、低买高卖和经营模拟玩法的玩家。
- 希望在小型社群或个人服务器中游玩的用户。

### 核心体验

1. 获得资金并选择购买补充包或直接从市场购卡。
2. 开包获得不同稀有度、属性和价值的卡牌。
3. 根据价格历史与市场信号决定买入、持有或卖出。
4. 在库存、现金流、收藏完成度和风险之间取得平衡。

## 2. 首发版本边界建议

首发版本建议优先实现单人游戏与系统驱动的市场，避免一开始就引入玩家间实时撮合、支付、聊天和复杂反作弊。这样可显著降低部署、运营与平衡成本；后续再按玩家数量扩展为多人市场。

## 5. 建议的数据实体

- 用户：账号、权限、余额、等级、创建时间。
- 卡牌系列：名称、主题、发售状态、市场参数。
- 卡牌：系列、稀有度、变体、基准价、当前价格、图片与说明。
- 补充包：价格、卡位、概率表、保底规则、可用时间。
- 玩家库存：用户、卡牌、数量、平均成本、品相（如启用）。
- 市场报价：卡牌、买入价、卖出价、更新时间、流动性。
- 价格历史：卡牌、时间点、价格、变化原因。
- 交易流水：用户、类型、数量、价格、手续费、前后余额。
- 开包记录：用户、补充包、消耗、产出卡牌、结果价值。
- 市场事件：作用范围、持续时间、影响系数、展示文案。
- 管理审计日志：操作人、对象、前后值、时间、来源。

## 6. 页面与导航草案

1. 首页 / 仪表盘：余额、净资产、今日盈亏、市场指数、待领取奖励。
2. 补充包商店：补充包列表、概率、价格、购买与开包入口。
3. 开包结果：卡牌揭示、重复处理、价值统计、再次开包。
4. 市场：搜索、筛选、买卖报价、价格曲线、新闻事件。
5. 我的库存：持仓、市值、收益率、批量出售、收藏入口。
6. 卡牌图鉴：系列进度、卡牌详情、未获得卡牌。
7. 任务 / 成就：目标、进度、奖励。
8. 经营报表：资金曲线、开包盈亏、交易盈亏、持仓分布。
9. 管理后台（管理员）：内容、概率、市场、玩家、事件和日志。

## 7. 验收标准示例

- 玩家能从新存档开始，获得初始资金，购买并开出补充包，且获得的卡牌进入库存。
- 玩家能按当前报价买入和卖出卡牌；余额、库存和交易记录保持一致。
- 至少能查看每张卡的当前价格与历史价格走势，并能看出涨跌。
- 市场价格会按既定周期更新，且事件能改变指定卡牌或系列的价格。
- 服务可根据部署文档在一台常见 Linux 服务器上启动，重启后数据仍存在。
- 管理员能在不改代码的前提下新增卡牌、补充包和市场事件。
