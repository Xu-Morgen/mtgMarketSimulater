无。

## 特殊点

- 权威边界：余额、净资产占位和账本均由 Fastify API 返回；前端只做展示格式化，不合计或结算金额。
- 幂等：创建 mutation 在单次用户意图的重试期间保留同一 `Idempotency-Key`；后端仍是存档与初始资金唯一性的最终权威。
- 数据隔离：用户私有 query key 包含 `userId`，并在认证切换时清空缓存；A 账户存档不会被 B 账户显示。
- 验证：`pnpm --filter @mtg-market/web check` 通过；I07F Playwright 用例在 Chromium 桌面与 390px 窄屏均通过。全量双项目 Playwright 会共享 localhost IP 并触发后端每分钟 10 次的认证限流，因此按项目独立启动执行 I07F 用例，未更改生产限流策略。
