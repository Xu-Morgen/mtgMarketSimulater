## 特殊点

- 权威边界：浏览器只读取本地 `/v1/admin/*` API 并提交意图；Scryfall 仅由受控后台任务访问，目录替换保持在 API/SQLite 权威层。
- 契约与安全：状态 DTO 不含外部下载 URL 或 Provider 原始响应；管理路由继续由服务端 admin RBAC 保护，普通玩家 API 返回 403。
- 外部 Provider：Scryfall 会拒绝 Node 默认 User-Agent；Bulk 和图片下载均通过仅服务端可配的应用标识发送，不暴露给浏览器。
- 数据兼容性与任务审计：迁移 `0009_job_run_attempts.sql` 为现有 SQLite 库添加运行记录指针；Bulk 客户端按 gzip 魔数解压，以兼容 Provider 的压缩传输。
- 幂等与审计：每个触发/重试意图生成幂等键，等待/运行状态禁用重复提交；任务 ID 可关联持久化任务和现有写路由审计记录。
- 路由安全：管理员登录时无条件忽略 URL 的 `next`，避免任意 URL 指令改变服务端角色对应的默认首页。
- 验证：通过 contracts/API/Web TypeScript 检查、目录 API Vitest 测试，以及 I09F 6 个 Chromium（桌面与窄屏）Playwright 用例。
