## 特殊点与已知限制

- refresh token 从不明文持久化；Cookie 仅作用于认证路径，生产环境额外启用 `Secure`。refresh token 被重放时会撤销其轮换后代，避免被盗旧凭据继续延展。
- 基础频率限制仅适用于当前单进程部署；将来横向扩容时必须替换为共享限流存储，不能把进程内 Map 当作跨实例风控。
- JWT 签名密钥必须从运行环境提供且至少 32 字符；运行时数据库和密钥文件均未修改或纳入源码。
