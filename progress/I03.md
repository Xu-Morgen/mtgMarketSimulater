## 特殊点

- 权威边界：数据库包只提供 schema、迁移和技术事务原语；余额、库存等业务决策仍由 API application 层编排，浏览器和 AI 没有写入权。
- 兼容性：`0002_jobs_evolution.sql` 支持 I03 前已有的仅含旧字段、且可含同类型历史任务的 `jobs` 原型库；其以任务 ID 回填唯一键后建立约束。新库依次运行两份迁移获得相同结构。
- 幂等与审计：已建立 `idempotency_requests`、`audit_logs`、`fact_events` 和 `outbox` 基础表及唯一约束，具体业务写入将在后续迭代接入。
- 安全与数据：金额表字段使用 SQLite `INTEGER` 并在入口拒绝浮点值；所有时间由 `utcNow()` 生成 UTC ISO 8601。每个迁移在单独事务中执行，失败不会留下部分 schema 或迁移标记。
- 验证：2026-07-24 已通过 `pnpm check` 与 `pnpm test`。
