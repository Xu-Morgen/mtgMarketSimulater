# 后端需求

本文件定义服务端的经济真相、事务和异步任务。完整技术栈见[技术栈与模块职责边界.md](技术栈与模块职责边界.md)。

## 1. 技术与运行要求

- Fastify + OpenAPI；SQLite（WAL 模式）+ Drizzle ORM；SQLite `jobs` 表与进程内任务循环；Docker Compose 部署。HTTP 层统一返回 contracts 响应包络、回传请求 ID、使用结构化 Pino 脱敏日志，并仅允许配置的 CORS Origin。
- 所有余额、库存、订单、保证金、开包、比赛奖励与成就写操作仅由服务端完成。
- 关键结算使用 SQLite 短事务、唯一约束、条件更新和 idempotency key；所有经济变动写追加式审计流水。单实例串行处理结算，避免 SQLite 写竞争。
- 数据库 schema 与 SQL 迁移统一由 `packages/database` 管理；启动先执行版本化迁移与 `PRAGMA integrity_check`。每个迁移必须短事务原子提交，失败时不写该迁移记录且保留迁移前状态。

## 2. 业务模块

| 市场与 NPC | 消费已结算的开包/NPC/P2P 事实事件，使用规则引擎计算参考价、供需聚合、系列/关联/事件系数、NPC 买卖价、价差与限额。 |
| 库存与开包 | 管理可用/锁定库存；开包、扣款、库存变化在同一事务完成。 |

## 2.1 I07B 存档、账户与账本（已完成，2026-07-24）

- `POST /v1/archive` 在用户、存档、账户、初始 `initial_funding` 流水、业务审计与幂等完成记录之间采用单个 SQLite 短事务；`game_archives.user_id`、账户币种与 `(actor_id, idempotency_key)` 唯一约束共同防止重复发放。
- 初始资金规则为 `initial-funds/v1`，从 `@mtg-market/rules` 解析，定义同步存入 `rule_versions`；每份存档保存使用的规则版本。余额只以 `GAME_CREDIT` 整数最小单位保存，数据库强制总额等于可用额加冻结额且不为负。
- users application 提供 reserve/release/capture 共享资金原语。冻结记录必须关联业务实体和原因；扣除必须由 active hold 发生并新增 debit 账本。任何未来订单或履约操作不得直接更新账户字段。

## 2.2 I10B 库存、成本与资产锁定（已完成，2026-07-24）

- `inventory_holdings` 以 `(user_id, sku_id)` 唯一记录持有量、可用量、订单锁定量、比赛锁定量、整数移动平均成本和可空市值快照；数据库约束强制持有量等于三类数量之和且全部非负。
- `inventory_holds` 保存订单/比赛的 active、released、captured 状态及关联实体，`inventory_entries` 为不可变数量与成本流水。锁定使用条件更新；释放或 capture 仅可消费 active hold，因此重放不会超额解锁或产生幽灵库存。
- `InventoryService.withLedgerTransaction` 明确要求开包、订单和比赛将库存、账本、事实事件及审计置于同一 SQLite 短事务。玩家 API 只提供当前用户的库存总览、单卡持仓与可分页对账查询；不提供直接改库存、成本或锁定量的接口。
| P2P 订单 | 双边委托确认、资金/库存锁定、撮合、模拟履约期限、保证金、取消与风控。 |
| 卡组与比赛 | 校验库存与赛制、锁定参赛卡牌、确定性赛果、奖励、成就与重放信息。 |
| 管理与审计 | 管理员仪表盘聚合；活动草稿、预览、发布/定时发布、暂停和结束；用户管理命令；参数版本化；只读、分页、脱敏的后台/经济/任务/异常交易/Agent 日志查询。 |

## 3. 异步任务

- `catalog.sync`：使用 Scryfall Bulk Data 同步卡池，按需写入本地卡图缓存；失败不删除现有目录或图片。
- `prices.sync`：每日同步 MTGJSON；成功后以快照唯一键投递 `market.reprice`，失败继续使用最近成功快照并告警。
- `daily.rollover`：按服务器自然日开放工作资金和刷新比赛，必须幂等。
- `market.reprice`：根据价格快照、供需事件、系列周期、关联卡、流动性与运营事件生成有界游戏报价；失败保留最近成功报价。
- `tournament.settle`：结算赛果、奖励与成就，成功后才投递 AI 战报任务。
- `narrative.generate`：调用 AI 或生成模板战报；绝不回滚比赛结果。
- `order.expire`：处理过期委托、模拟履约时限和保证金。
- `backup.create`：生成并校验 SQLite 一致性备份，执行保留策略；失败不得删除最近成功备份。

### I09B 目录同步实现（已完成，2026-07-24）

- `catalog.sync` 仅由服务端任务处理器调用 Scryfall Bulk Data。它使用服务端 `SCRYFALL_USER_AGENT`（不得使用 Node 默认值）请求元数据、Bulk 文件和卡图，兼容 gzip 编码，并按对象扫描完整数组、只保留启用系列，避免完整 Bulk 文件超出 Node 字符串上限；它记录来源版本、下载 URI、SHA-256、启用系列、导入差异、运行时间和失败摘要；`catalog_sync_state` 永远指向最近一次成功运行。
- 导入先验证下载 JSON、必要印刷字段、Scryfall UUID、支持的工艺和重复印刷；随后在单个 SQLite 短事务内替换 `scryfall` 来源的系列、印刷、SKU 和图片元数据。任一失败不删除上次成功目录或状态指针。
- 仅 payload 中明确指定的卡图会下载到 `CATALOG_DATA_DIR/images`；文件名受 UUID/扩展名白名单约束，玩家只经认证后的本地静态路由读取，浏览器不访问 Scryfall。
- `/v1/admin/catalog/sync` 的读写都要求 `admin` 角色；写入要求 `Idempotency-Key` 并以任务唯一键重放同一次投递。固定夹具覆盖 checksum、截断、重复、缺字段、非法图片路径和事务中断。

### I05 调度语义

- 任务状态仅可按 `pending/failed → running → succeeded|failed|dead` 转换；`running` 必须带有限期租约。过期任务在启动补跑或领取前回收，已达最大尝试次数的任务转为 `dead`。
- 同一 `(type, unique_key)` 只保留一个任务事实；每一次领取以全局单调的运行序号写 `job_runs` 运行记录，错误摘要截断保存。失败任务使用有上限的指数退避；`failed` 和 `dead` 可通过管理接口人工重试，并重置调度尝试计数，但不重用历史运行记录序号。
- 处理器由任务注册表预先注册；I05 固定预注册 `catalog.sync`、`prices.sync`、`daily.rollover`、`market.reprice`、`tournament.settle`、`order.expire`、`narrative.generate`、`backup.create`。业务实现必须通过各自 application 用例并以自身事实唯一约束保证结果至多一次。

## 4. 不变量

1. 不允许负余额、负库存或同一张卡同时被订单与比赛使用。
2. 无外部价、零价、映射不完整或同步异常 SKU 暂停新增交易；已有库存按最后成功快照估值。
3. 取消模拟履约的保证金扣款、订单状态、库存/买方资金恢复和流水必须同一事务、可追溯。
4. 外部快照、开包记录、赛果、奖励与审计日志只追加，不被后续任务覆盖。
5. 密码仅保存 Argon2id 哈希；access token 必须短期有效，refresh token 仅以哈希持久化并可撤销。Cookie 型刷新与注销必须校验 CSRF，admin 路由必须在服务端复核角色。
6. 管理员修正余额或库存必须调用所属模块的补偿命令，在同一短事务写新流水、原因、操作者、幂等键和关联原记录；禁止直接覆盖最终值、删除或改写历史流水。
7. 已发布活动不可原地改写；发布、暂停和结束采用实体版本与幂等键，活动的时间、作用范围和影响参数必须经服务端规则校验，成功后以活动版本唯一键投递可重放的 `market.reprice`。
8. 审计与运行日志不提供删除或通用修改接口；查询必须服务端分页、鉴权和脱敏，不返回密码哈希、令牌摘要、Cookie、密钥或未处理的 Provider 敏感输入。

## 5. 管理后台服务端能力（I30B）

- 所有管理读写路由位于 `/v1/admin/*`，先验证有效会话再要求 `admin` 角色；前端路由保护不能替代此检查。管理写请求要求 `Idempotency-Key`，并记录请求 ID、操作者、对象、版本、原因和结果摘要。
- 活动 API 区分草稿保存、发布预览、发布/定时发布、暂停和结束命令。预览返回可确认的版本、目标范围、UTC 生效区间、参数上限/冲突和预计任务；提交时版本不一致必须返回稳定冲突错误。
- 用户查询按用户 ID、用户名、角色和状态服务端分页；详情只返回完成管理任务所需的账户/会话状态与余额、账本、库存、订单、比赛、审计摘要。冻结/解冻、会话撤销和补偿修正分别使用显式命令，不提供跨表通用 CRUD。
- 日志查询按时间、操作者、用户、实体、动作、结果、请求 ID、任务类型分页筛选，并返回脱敏详情和关联标识。管理页面查看日志本身可记录访问审计，但不得制造递归的无限日志链。
- Fastify inject + 临时 SQLite 集成测试覆盖普通玩家 403、会话撤销、重复管理命令、同键异参、活动版本冲突/参数越界/重复发布、补偿事务回滚、日志不可修改和敏感字段脱敏。
